## 基本概念

### 什麼是「提交」（Commit）？

提交是保存您的檔案變更的快照。每次提交都會記錄：
- 修改了哪些檔案
- 修改的內容
- 修改的時間
- 修改的原因（提交信息）

### 什麼是「推送」（Push）？

推送是將您的提交上傳到 GitHub 伺服器。只有推送後，其他人才能看到您的變更。

### 工作流程

```
修改檔案 → 提交（Commit） → 推送（Push） → 上傳到 GitHub
```

## 第一步：檢查變更

### 1.1 打開 GitHub Desktop

1. 雙擊 GitHub Desktop 應用圖標
2. 如果已經打開，跳到下一步

### 1.2 查看「Changes」標籤

1. 在 GitHub Desktop 左側，您應該看到幾個標籤
2. 點擊「Changes」標籤（變更）
3. 您應該看到所有修改的檔案列表

### 1.3 檢查修改的檔案

在「Changes」標籤中，您應該看到：

| 欄位 | 說明 |
|------|------|
| 檔案名稱 | 修改的檔案路徑 |
| 綠色「+」 | 新增的檔案 |
| 黃色「M」 | 修改的檔案 |
| 紅色「-」 | 刪除的檔案 |

### 1.4 查看具體修改

1. 在檔案列表中點擊一個檔案
2. 右側會顯示該檔案的具體修改
3. 綠色部分 = 新增的內容
4. 紅色部分 = 刪除的內容

**示例：**
```
- 舊的構建命令：pnpm build
+ 新的構建命令：npm run build
```

## 第二步：提交變更

### 2.1 確認所有檔案都被選中

在「Changes」標籤中，每個檔案左側應該有一個 ✓ 複選框。

**確保所有要上傳的檔案都被勾選。**

### 2.2 取消選擇不需要的檔案（可選）

如果您不想上傳某個檔案：
1. 點擊該檔案左側的 ✓ 複選框
2. 複選框會變成空的，表示該檔案不會被提交

### 2.3 填寫提交信息

在 GitHub Desktop 的左下角，您應該看到：

```
Summary (必填)
[輸入框：簡短的變更說明]

Description (可選)
[輸入框：詳細的變更說明]
```

### 2.4 輸入提交信息

在「Summary」欄位中輸入簡短的說明。例如：

```
Fix: Update GitHub Actions pnpm configuration
```

或者：

```
Update deploy workflow to use pnpm v3
```

### 2.5 點擊「Commit」按鈕

1. 在左下角找到「Commit to main」按鈕
2. 點擊它
3. 等待提交完成（通常只需 1-2 秒）

✅ **完成！您的變更已提交到本地。**

---

## 第三步：推送到 GitHub

### 3.1 檢查「Push」按鈕

提交完成後，您應該看到：
- 頁面頂部出現「Push origin」按鈕
- 或者在右上角看到一個帶有向上箭頭的按鈕

### 3.2 點擊「Push origin」

1. 點擊「Push origin」按鈕
2. 等待推送完成（通常需要 5-10 秒）
3. 按鈕會消失，表示推送成功

✅ **完成！您的變更已上傳到 GitHub。**

## 驗證上傳成功

### 4.1 檢查 GitHub 網站

1. 打開瀏覽器
2. 進入 `https://github.com/your-username/chunan-victory-church`
3. 您應該看到：
   - ✅ 最新的提交信息顯示在頁面上
   - ✅ 時間戳顯示為「just now」或「a few seconds ago」
   - ✅ 修改的檔案已更新

### 4.2 檢查 GitHub Actions

1. 在 GitHub Repository 頁面中
2. 點擊「Actions」標籤
3. 您應該看到：
   - ✅ 最新的工作流運行已開始
   - ✅ 狀態顯示為「In progress」或「Success」

### 4.3 等待部署完成

1. 點擊最新的工作流運行
2. 等待所有步驟完成（通常需要 10-15 分鐘）
3. 最終狀態應該顯示為「Success」（綠色對勾）

✅ **如果所有步驟都成功，說明您的變更已成功部署！**

## 常見問題

### Q1：我看不到「Changes」標籤？

**A：** 
1. 確保 GitHub Desktop 已打開
2. 確保您已選擇了正確的 Repository（`chunan-victory-church`）
3. 如果還是看不到，嘗試重啟 GitHub Desktop

### Q2：提交後沒看到「Push origin」按鈕？

**A：** 
1. 檢查頁面頂部或右上角
2. 有時按鈕會在不同的位置
3. 如果還是找不到，嘗試重啟 GitHub Desktop

### Q3：推送時出現錯誤？

**A：** 
1. 確保您已登入 GitHub Desktop
2. 確保您有 Repository 的寫入權限
3. 嘗試重新登入 GitHub Desktop
4. 檢查網路連接

### Q4：如何撤銷已提交的變更？

**A：** 
1. 如果還沒推送，點擊「History」標籤
2. 右鍵點擊最新的提交
3. 選擇「Revert this commit」
4. 然後推送

### Q5：如何查看提交歷史？

**A：** 
1. 點擊「History」標籤
2. 您應該看到所有之前的提交
3. 點擊任何提交可以查看詳細信息

### Q6：行結尾警告（CRLF vs LF）是什麼？

**A：** 
1. 這是檔案格式的差異
2. 不會影響功能
3. GitHub 會自動處理
4. 您可以安全地接受這個變更
